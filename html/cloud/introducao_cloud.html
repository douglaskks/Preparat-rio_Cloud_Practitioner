Conceitos do Cloud Computing

Vamos iniciar falando sobre os conceitos do Cloud Computing, existe uma difinição que é bem conhecida onde ela diz que Cloud Computing é a entrega sob demanda de recursos de computação, banco de dados, armazenamento, aplicações ou qualquer outros recurso de tecnologia que é entregue através de uma plataforma via internet, onde o pagamento e preço é baseado no consumo.

Então pensando dessa forma, podemos entender que uma plataforma de Cloud Computing deve ter pelo menos esses três elementos que seria ser Sob demanda, ser uma Plataforma via internet e o pagamento dever ser contabilizado através do consumo ela ser considerada uma plataforma de Cloud Computing

Vantagens do Cloud Computing

Mudança na modalidade de gastos

Muda da modalidade de Despesa de capital, ou despesa de Aquisição de bens, CAPEx para o modelo de variável ou despesa operacional - OPEx.

Economia de Escala

A Amazon Web Services tem um volume de compras em uma escala muito grande, o que acaba beneficiando seus usuários, pois, como consequência temos uma diminuição de custo significativa com isso.

Capacidade

Com Cloud computing você cresce ou diminui capacidade necessária para atender seu negócio pagando apenas o que consumir ao decorrer do tempo. Quem trabalha ou já trabalhou com datacenters é um bom exemplo, pois é bem difícil prever o comportamento de ambas os lados, tanto da estrutura quanto dos usuários, pois os usuários podem aumentar de mais e a estrutura estiver abaixo do que precisa ou vice-versa.

Agilidade e Velocidade

O Cloud Computing permite que as empresas crie produtos e negócios em muito com pouco tempo, acelerando o time-to-market e melhorando a experiência do cliente. Assim os recursos estão disponíveis imediatamente na hora que o cliente ou a empresa quer.

Economia

Com o Cloud Computing as empresas deixam de gastar dinheiro para comprar e manter datacenters e foca no que realmente é importante para o negócio.

Global em poucos minutos

Um dos pontos que podemos achar mais interessante, óbviamnte sem diminuir as outras vanatgens é a capacidade das empresas terem recursos disponíveis globalmente podendo assim chegar a um número grande de regiões em pouquíssimo tempo e com baixíssima latência com baixo custo, melhorando assim a experiência do cliente.

Tipos de Cloud Computing

IaaS - Infraestrutura como Serviço

  • O contratante gerencia os servidores, sejam físicas ou virtuais
  • O contratante gerencia os sistemas operacionais e softwares adicionais
  • O Datacenter não tem responsabilidade sobre o que você faz com osrecursos (Serveless)
  • Empresas em geral compram serviços de hospedagem.

PaaS - Plataforma como Serviço

  • O Datacenter é responsável pelos recursos físicos ou virtuais, sistemas operacionais, softwares e alguns intens de segurança.
  • O contratante é responsável é responsável pela aplicação
  • Se você quiser colocar um website no ar, a empresa te prever todo o ambiente, bastando você fazer o upload da sua aplicação e configurá-la.

SaaS - Software como Serviço

  • Nesse modelo utiliza apenas o serviço e não se preocupa com nada, exemplos são o Gmail, Udemy Microsoft Office e Whatsapp.

Tipos de Instalação de Clou computing

Public Cloud

  • Amazon Web Services (AWS), Microsoft Azure, Google Cloud Platform

Hybrid Cloud

  • Combinação entre publica e privada.

Private Cloud

  • Geralmente é uma Instalação física em um Datacenter onde você faz todo o gerenciamento, usando diversas tecnologias, entre elas VMWARE e OpenStack.
